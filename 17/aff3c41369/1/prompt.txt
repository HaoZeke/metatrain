# Full-Graph FX Compilation for PET

## Context

PET's current `compile: true` compiles individual submodules (GNN layers, heads) separately with `torch.compile`. This causes:
1. Recompilation on every new tensor shape (no `dynamic=True` — fixed in earlier commit)
2. Many compiled/eager boundary crossings (each submodule is a separate compiled unit)
3. For force training, only the backward is compiled (forward is eager)
4. No cross-module optimization (kernel fusion can't span submodule boundaries)

PhACE (PR #921) and UPET (janus-share) both compile the **full model as a single FX graph** using `make_fx` + `torch.compile`. This gives maximum kernel fusion, zero boundary crossings, and includes force computation in the graph (eliminating double backward). We adopt the same pattern for PET.

## Approach

Follow the UPET/PhACE pattern:

1. Create a traceable forward function that takes batch tensors + explicit model params/buffers
2. Include force computation via `autograd.grad(create_graph=False)` inside the traced function
3. Trace with `make_fx(tracing_mode="symbolic")` — this decomposes autograd.grad into primitive backward ops, producing a flat FX graph
4. Compile with `torch.compile(dynamic=True, fullgraph=True, mode="max-autotune")`
5. Training's `loss.backward()` is now single backward through the FX graph (no double backward)

### Key benefit: SDPA everywhere
Since forces are computed inside the FX graph via `autograd.grad(create_graph=False)`, there is no double backward. This means PET can always use `scaled_dot_product_attention` (faster) instead of `manual_attention` in the compiled path.

**`_make_pet_compiled_forward()`** — Creates the traceable function (like UPET's `_make_forward_full_nl`):

```python
def _make_pet_compiled_forward(batch_model, param_names, buffer_names,
                                target_names, output_shapes,
                                compute_forces, compute_stress):
    n_params = len(param_names)

    def forward_fn(edge_vectors, element_indices_nodes, element_indices_neighbors,
                   padding_mask, reverse_neighbor_index, cutoff_factors,
                   system_indices, neighbor_atom_indices, n_structures,
                   *params_and_buffers):
        params = dict(zip(param_names, params_and_buffers[:n_params]))
        buffers = dict(zip(buffer_names, params_and_buffers[n_params:]))

        # Compute edge_distances inside compiled graph (differentiable)
        edge_distances = torch.sqrt((edge_vectors ** 2).sum(-1) + 1e-15)

        # Call model forward via functional_call
        raw_predictions = functional_call(
            batch_model, (params, buffers),
            (element_indices_nodes, element_indices_neighbors,
             edge_vectors, edge_distances, padding_mask,
             reverse_neighbor_index, cutoff_factors)
        )

        # Aggregate per-atom → per-structure energy (for energy targets)
        n_atoms = edge_vectors.shape[0]
        n_struct = n_structures + 1  # +1 padding for scatter
        energy = torch.zeros(n_struct, dtype=edge_vectors.dtype, device=edge_vectors.device)
        per_atom_energy = raw_predictions[energy_target_name][energy_block_key]
        energy.scatter_add_(0, system_indices, per_atom_energy.squeeze(-1))

        forces = None
        stress = None

        if compute_forces or compute_stress:
            (dE_dR,) = torch.autograd.grad(
                energy[:n_structures].sum(), edge_vectors, create_graph=False
            )
            dE_dR = dE_dR * padding_mask[:, :, None].float()

            if compute_forces:
                # d(E)/d(pos[i]) = -Σ_j dE_dR[i,j] + Σ_{(k,j):neighbor=i} dE_dR[k,j]
                grad_as_center = -dE_dR.sum(dim=1)  # [n_atoms, 3]
                flat_dE = dE_dR.reshape(-1, 3)
                flat_idx = neighbor_atom_indices.reshape(-1, 1).expand(-1, 3).long()
                grad_as_neighbor = torch.zeros(n_atoms, 3, ...)
                grad_as_neighbor.scatter_add_(0, flat_idx, flat_dE)
                forces = grad_as_center + grad_as_neighbor  # = d(E)/d(pos)

            if compute_stress:
                # Virial: σ = (1/V) Σ r ⊗ (dE/dr)
                virial_per_atom = torch.einsum('ema,emb->eab', edge_vectors, dE_dR)
                stress = torch.zeros(n_struct, 3, 3, ...)
                stress.scatter_add_(0,
                    system_indices[:, None, None].expand(-1, 3, 3),
                    virial_per_atom)
                stress = stress[:n_structures]

        return energy[:n_structures], forces, stress, raw_predictions

    return forward_fn
```

**`compile_pet_model(model, dataloader, ...)`** — Traces and compiles:

```python
def compile_pet_model(model, dataloader, compute_forces, compute_stress):
    batch_model = _PETBatchForward(model)
    replace_silu_modules(batch_model)

    params = dict(batch_model.named_parameters())
    buffers = dict(batch_model.named_buffers())
    param_names, buffer_names = list(params.keys()), list(buffers.keys())

    forward_fn = _make_pet_compiled_forward(
        batch_model, param_names, buffer_names,
        model.target_names, model.output_shapes,
        compute_forces, compute_stress
    )

    # Get sample batch for tracing
    batch = next(iter(dataloader))
    systems, targets, _ = unpack_batch(batch)
    batch_tensors = systems_to_batch(systems, model.requested_nl, ...)

    old_duck = torch.fx.experimental._config.use_duck_shape
    torch.fx.experimental._config.use_duck_shape = False
    try:
        fx_graph = make_fx(
            forward_fn, tracing_mode="symbolic",
            _allow_non_fake_inputs=True,
        )(
            batch_tensors.edge_vectors.clone().requires_grad_(True),
            ...,
            *list(params.values()), *list(buffers.values()),
        )
    finally:
        torch.fx.experimental._config.use_duck_shape = old_duck

    compiled = torch.compile(fx_graph, dynamic=True, fullgraph=True, mode="max-autotune")
    return compiled, param_names, buffer_names
```
